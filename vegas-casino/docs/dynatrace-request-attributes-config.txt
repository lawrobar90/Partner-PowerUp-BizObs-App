# Dynatrace Request Attributes Configuration for LoadRunner Integration
# 
# To set up these request attributes in Dynatrace:
# 1. Go to Settings → Server-side service monitoring → Request attributes
# 2. Create a new request attribute for each of the following configurations

## Request Attribute 1: Test Step Name (TSN)
Name: LoadRunner Test Step Name
Data source: HTTP request header
Header name: x-dynatrace-test
Extract value using regex: TSN=([^;]+)
Description: Identifies the specific test step within the LoadRunner script

## Request Attribute 2: Load Script Name (LSN)  
Name: LoadRunner Script Name
Data source: HTTP request header
Header name: x-dynatrace-test
Extract value using regex: LSN=([^;]+)
Description: Name of the LoadRunner script being executed

## Request Attribute 3: Load Test Name (LTN)
Name: LoadRunner Test Name  
Data source: HTTP request header
Header name: x-dynatrace-test
Extract value using regex: LTN=([^;]+)
Description: Unique identifier for the load test execution

## Request Attribute 4: Virtual User ID (VU)
Name: LoadRunner Virtual User
Data source: HTTP request header  
Header name: x-dynatrace-test
Extract value using regex: VU=([^;]+)
Description: ID of the virtual user making the request

## Request Attribute 5: Source ID (SI)
Name: LoadRunner Source ID
Data source: HTTP request header
Header name: x-dynatrace-test  
Extract value using regex: SI=([^;]+)
Description: Identifies LoadRunner as the source of requests

# Example header value:
# x-dynatrace-test: TSN=Slot_Spin;LSN=Vegas-Slots-Load-Test;LTN=Vegas_Slots_5Min_Load_Test_20250929_143022;VU=3;SI=LoadRunner

# After configuring these request attributes in Dynatrace, you can:
# - Filter services by these attributes
# - Create custom dashboards showing load test metrics
# - Analyze performance by test step or virtual user
# - Set up automated alerting during load tests
